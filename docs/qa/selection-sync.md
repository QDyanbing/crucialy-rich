# QA：选区双向同步（第一版）

## 范围

验证浏览器折叠选区和模型 `RangeSelection` 的双向同步。

## 自动化测试

- `packages/core/tests/render/selection-sync.test.ts`：读取折叠浏览器选区、生成 DOM 范围、恢复浏览器选区。
- `tests/e2e/demo-shell.spec.ts`：演示渲染区选区同步到调试面板。

命令：

```sh
pnpm test
pnpm test:e2e
```

## 手测场景

| 场景         | 操作                       | 期望                      | 结果 |
| ------------ | -------------------------- | ------------------------- | ---- |
| 读取折叠光标 | 在渲染文本中设置浏览器光标 | 返回同位置模型选区        | 通过 |
| 恢复折叠光标 | 传入折叠模型选区           | 浏览器选区落到对应 DOM 点 | 通过 |
| 演示选区同步 | 在渲染区改变浏览器选区     | 选区 JSON 同步更新        | 通过 |
| 无选区范围   | 清空浏览器选区后读取       | 返回 `undefined`          | 通过 |

## 当前限制

- 当前自动化只覆盖折叠选区。
- 不保留反向选区方向。
- 不处理编辑输入、IME、删除或换行。

## 结论

选区双向同步第一版已闭环，可以进入 React 组件 API 接入阶段。
